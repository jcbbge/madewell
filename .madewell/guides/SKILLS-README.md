# Skills

**What:** Thinking tools your guide uses when the situation calls for them.
**When:** You don't invoke them directly — your guide reads them when relevant.
**How:** Give your guide a problem. It reaches for the right lens.

---

## The Skills

| Skill | What it does |
|---|---|
| `luck` | The Oracle. Reveals hidden patterns, warmth trails, what's trying to emerge. Always include when stuck. |
| `criticality` | Reads the cognitive state of the work. Subcritical (stuck, need more)? Supercritical (overwhelmed, prune)? |
| `discovery` | Processes unstructured thinking into actionable insights. For brain dumps and first sessions. |
| `systems-thinking` | Maps feedback loops, leverage points, second-order effects. For complex, interconnected problems. |
| `reframing` | Challenges how the problem is framed. Finds the actual problem worth solving. |
| `blind-spots` | Surfaces what isn't being considered. Before major decisions. |
| `challenging-assumptions` | Steel-mans an idea then systematically finds failure modes. |
| `exploring-possibilities` | Divergent thinking. Generates directions before converging. |
| `debug-hypothesis` | Scientific debugging: observe, hypothesize, experiment, conclude. Never bulldoze. |

---

## The Power Move

When you're stuck on something hard:

```
I'm stuck on [describe the problem].
Load luck, [relevant skill], and one other skill that might help.
Analyze this through each lens and tell me what you see.
```

Each skill is a different way of seeing the same thing.
The insight almost always comes from the one you didn't expect.
