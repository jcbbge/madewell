# Exploring Possibilities

**When to use:** Early brainstorming, creative blocks, "what's possible here?", "what am I not seeing?", generating novel connections.

---

## Core Principles

### 1. Divergent First
Generate multiple possibilities before converging. Defer judgment. Prioritize quantity over quality initially.

### 2. Cross-Domain Promiscuity
Freely pull patterns from unrelated fields. Identify structural similarities, not surface features.

### 3. Embrace Uncertainty
Explore low-confidence ideas that might be valuable. Label speculation explicitly.

### 4. Pattern Hunting
Look for structural similarities across domains. Extract functional patterns, not implementations.

### 5. Question Assumptions
Challenge constraints that might not be real. Ask "what if the opposite were true?"

---

## Key Techniques

### Divergent Burst

Rapid generation without evaluation:
1. Set quantity goal ("10 different angles")
2. Defer judgment explicitly
3. Use prompts: "What else?", "Different angle?", "Opposite approach?"
4. Present all options before discussing any

**Example:**
```
"Let's explore 8 framings before converging:

1. Narrative Control Tool — borrowing from theater lighting
2. Data Visualization Layer — financial trading floor patterns
3. Attention Orchestration — theme park experience design
4. Real-time Feedback System — sports broadcast production
5. Audience Participation — interactive TV mechanics
6. Biometric-Responsive — health monitoring dashboards
7. Adaptive Performance — video game difficulty adjustment
8. Multi-sensory Experience — VR environment design

Which 2-3 feel most promising?"
```

---

### Cross-Domain Bridge Building

1. Identify target domain challenge
2. Find source domains with analogous challenges
3. Extract transferable pattern
4. Map to target domain
5. Adapt and refine

**Example:**
```
Fire safety = mandatory recurring service + regulatory compliance

Source domains with same structure:
- Elevator inspections → IoT predictive maintenance
- Pest control → subscription model innovation
- HVAC maintenance → software platform aggregation

Which pattern transfers?
```

---

### Provocation & Challenge

Deliberately provocative statements to break mental patterns:

**Provocation operators:**
- **Reversal:** "What if we did the opposite?"
- **Exaggeration:** "What if it were 100x bigger/smaller?"
- **Escape:** "What if constraint X didn't exist?"
- **Distortion:** "What if the sequence were reversed?"

**Example:**
```
Provocation: What if we had NO onboarding at all?

This forces us to ask:
- Why do we assume users need onboarding?
- What if the app were so intuitive it's unnecessary?
- Could we learn from apps with zero onboarding?

Useful extraction:
- Maybe onboarding compensates for poor UX
- What if we fixed UX instead of adding onboarding?
```

---

### Semantic Distance Maximization

Prioritize conceptually distant ideas over obvious ones:

**Standard ideas (low distance):**
- Better Gantt charts
- Slack integration
- More reporting

**High distance ideas:**
- Orchestra conducting: real-time coordination without communication
- Air traffic control: managing dependencies with precision
- Improv theater: structured flexibility and "yes, and"

---

### Assumption Inversion

1. List implicit assumptions
2. For each: "What if opposite were true?"
3. Explore implications
4. Identify valuable insights

**Example:**
```
Assumption: "We need a mobile app"

Inversions:
- What if NO app, just mobile web?
- What if SMS-based instead?
- What if we partner with existing app?
- What if service is asynchronous?
```

## What This Surfaces

- **Unexpected analogies** between unrelated domains
- **Edge cases as opportunities**
- **Assumptions to challenge**
- **Second-order effects**
- **Alternative problem formulations**
- **Novel combinations**

---

## When to Stop

- User asks for specific solution
- Time-sensitive decision needed
- Same patterns repeating without new insight
- User shows frustration with divergence

**Transition:** "That one, go deeper" → shift from exploring to executing

---

## Key Reminder

> Exploration is for opening, not closing. Generate multiple directions. Follow the promising threads. Then converge deliberately.
