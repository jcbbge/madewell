2026-05-30 | validation is the fourth atomic phase of all work, executed by independent sub-agents | every initiative runs Ideate → Plan → Implement → Validate. The builder never validates their own work — structural separation between builder and verifier is the only defense against self-marking success. Validation criteria are pluggable "jump packs": coding has one, sales/marketing/ops would each have their own. The four-role shape (Builder, Designer, Runner, Triage) is universal; the verification content swaps per domain.
2026-05-30 | coding jump pack — independent test-design, test-run, and failure-triage sub-agents | an agent that writes its own tests will fudge them to pass. Test-Designer runs in parallel with Implementer from the same brief. Test-Runner is read-only on code/tests. Failure-Triage is a fresh agent that classifies failures as (a) pre-existing bug, (b) wrong implementation, or (c) wrong test. Applies to code-generating work only; non-code briefs declare Testing: Applies: no with reason.

2026-06-20 | Made Well and Rumen relate as host↔organ, not as a stack | neither is the other's foundation: Made Well runs without Rumen, and Rumen runs naked in Arc with no Made Well — two things that each stand alone can't be foundation-and-superstructure. Made Well is the HOST (the work loop + persistence + the Guide persona); Rumen is the quality ORGAN it ships pre-installed and can run without. They meet at exactly two touchpoints — Verify (check work against the walls/assays) and Land (hand over proposed−accepted as cud). Rumen is the PRO-PACK implementation of Made Well's quality shape: the Enforcer skill is the batteries-included default behind the SAME contract. The dependency is one-way and optional — Made Well → (optional) → Rumen across the contract only; Made Well never imports Rumen, never writes to .rumen/, never breaks when it's absent. Resolves the contradiction where Rumen's PACK-CONTRACT §5 made the engine the foundation and Made Well's MERGE-INVENTORY made it a component — it is neither; it is a transplantable organ. This closes the former R15 engine↔kernel seam: the seam IS the quality contract, not a wrapping order.
2026-06-20 | the unit of work has a FOUR-leg lifecycle: Discovery → Commit → Cycle → Land, wrapping the four-phase execution core | the four phases (Imagine→Plan→Make→Verify) are the skeleton/core ("Cycle"), fractal and recurring at every altitude. Around them: Discovery (divergent intake — "what deserves attention"), Commit (convergent admission gate — "this, not that, now"; the WIP limiter), Land (the outlet — Ship the DELTA outward + Reflect the nutrient inward; carries single-loop feedback→Discovery and double-loop feedback→process). The four legs ALTERNATE anabolic↔catabolic (take-in / cut-down / build / release), which is why four is stable — opposite legs balance like a table; the old system was a two-legged table (all intake, no release) and fell over. Discovery and Land are boundary membranes (an adjoint pair — the formal reason consume/produce must balance); Commit is the newly-named admission gate (the catabolic-convergent face that was missing); the Cycle is the core. "Pillars" is freed and reserved for the dev-pack's four software DOMAINS (backend/frontend/api/ci-cd) — a different axis (what you build, not when).
2026-06-20 | catabolism is the missing primitive across the whole system; Rumen is its reference implementation | every layer has an anabolic intake but no catabolic release, and that single asymmetry is the root of every flooding/backup hit so far: work has Discovery but no Land (WIP floods), attention has capture but no Commit-gate (active-list thrash), conventions had promotion but no apoptosis (the rule pile — SOLVED in Rumen), memory has capture but no forgetting (context rot). One disease, many masks. Catabolism is NOT destruction — it is the controlled breakdown that returns nutrients and frees capacity (ship+drain, say-no/descope, forget, slough are the same operation in different substrates). Rumen's mechanic is the TEMPLATE every layer copies: earn your place by economics, die when you stop paying, sunset-review to distinguish "internalized" from "still-needed-but-silent." KERNEL LAW: every contract in the lattice must expose a catabolic method, modeled on Rumen.
2026-06-20 | Made Well is a contract lattice, not a feature set | the kernel = a small set of irreducible contracts (Intake/Discovery, Commit, Cycle, Outlet/Land, Memory, Quality, Persona) + the cycle topology that wires them + the dumbest-possible default implementation of each. Everything else — the Guide persona, the dev pack, Rumen, a memory backend, the thinking-tools — is a swappable implementation behind a contract. The kernel depends ONLY on contracts, never on a specific implementation; plug-and-play (swap my opinionated default for someone else's) falls out for free. Every shape therefore has the same four-slot structure: contract · batteries-included default · my pro pack · open-ecosystem slot. Persona-as-module, host↔organ (Rumen), and the memory-substrate question are all the same move applied to different rows.
